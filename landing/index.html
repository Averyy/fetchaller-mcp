Quick Setup (2 minutes)

Terminal 
# Clone and install
git clone https://github.com/Averyy/fetchaller-mcp.git
cd fetchaller-mcp
npm install

# Add to Claude Code
claude mcp add fetchaller -- node /path/to/fetchaller-mcp/index.js
~/.claude/settings.json 
{
  "permissions": {
    "allow": ["mcp__fetchaller__fetch"]
  }
}
CLAUDE.md (recommended) 
## Tool Selection

**DO NOT use `WebFetch`** - use fetchaller instead (no domain restrictions, no permission prompts).

**fetchaller replaces `WebFetch`, not dedicated MCPs.** If a dedicated MCP exists for a service (GitHub, Slack, etc.), use that MCP instead. Use fetchaller for general web fetching.

Restart Claude Code after setup. That's it!

FAQ

Why not just use WebFetch?

WebFetch requires you to approve each domain individually, and Reddit is completely blocked. fetchaller removes these limitations with a single permission rule.

Does Reddit work?

Yes! fetchaller automatically uses old.reddit.com for 65-70% token savings compared to new Reddit. Posts, comments, subreddits, and user profiles all work.

Is this safe?

Yes. fetchaller only does GET requests (read-only). It doesn't store cookies, maintain sessions, or write anything. It's the same as opening a URL in your browser.

What about curl?

Use curl for POST/PUT/DELETE requests, debugging headers, or when you need raw HTTP details. Use fetchaller for reading web pages and APIs during research.

Does it work with other MCP clients?

Yes! fetchaller is a standard MCP server. It works with Claude Desktop, Cursor, or any MCP-compatible client.
